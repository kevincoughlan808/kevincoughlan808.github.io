Kept the C++ momentum going this week. Pushed through Days 8–11 of the 100 Days of Code challenge.

## What I built

Days 8 and 9 covered file I/O with `ifstream`/`ofstream` and then namespaces, header files, and setting up a multi-file project with CMake. Day 9 felt like a big step — going from single-file exercises to actually splitting things across headers and source files made it feel more like real development.

Day 10 was the first proper milestone: a CLI calculator. Nothing groundbreaking, but it pulled together everything from the previous days into one working thing. Felt good to build something instead of just doing isolated exercises.

Day 11 moved into classes — constructors, initialiser lists, methods. Still early but I can already see how C++ wants you to think about object layout and initialization in a way Python just doesn't care about.

## What I learned

- CMake is its own beast. I spent more time fighting the build config than writing actual C++ on Day 9. But now that it's set up I get why people use it.
- Initialiser lists vs assigning in the constructor body — one of those things that seems like a nitpick until you understand it actually matters for performance and const members.
- Having a milestone project (the calculator) between concept days breaks up the grind nicely. I should plan more of those.

## Next up

- Keep going with OOP — inheritance, polymorphism, virtual functions are probably next.
- Refactor the calculator to use classes now that I've learned them.
