Didn't get as much done as I wanted this week, but I made progress on the front end and poked at the storage problem.

## What I built

Put together a basic single-page front end that hits the `/history` endpoint and renders a price chart using Chart.js. It's just an `index.html` file with some inline JavaScript — no build step, no framework. The chart shows the last 24 hours of BTC prices with timestamps on the x-axis. It looks rough but it works and it's weirdly satisfying to see the data visualized instead of staring at JSON.

I served it locally with Python's `http.server` for now. Haven't figured out where I want to host it yet.

## What I learned

- Chart.js is easy to get started with but the time axis formatting took me longer than I expected. The docs are fine, I just kept second-guessing the config options instead of just trying them.
- I looked into Railway volumes for persistent storage. The setup seems straightforward but I'd need to move off the free tier. Also spent some time reading about Supabase's free Postgres offering as an alternative. Haven't committed to either yet — I keep going back and forth, which is the same overthinking pattern from the caching decision a few weeks ago. Need to just pick one.

## Next up

- Hook the front end up to the live Railway API instead of localhost and deploy it somewhere — GitHub Pages would be easy since I already have it set up for this blog.
- Actually pick a persistent database solution and migrate the history endpoint. Leaning Supabase.
