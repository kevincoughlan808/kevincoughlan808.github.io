Another solid week — pushed through Days 39–43 of the 100 Days of Code challenge. Finally got into STL territory and then pivoted into data structures.

## What I built

Day 39 was ranges and views — `views::filter`, `views::transform`, `views::take`, and lazy pipelines. After weeks of templates and lambdas, ranges felt like the payoff — suddenly all that stuff clicks together. Day 40 was the Phase 4 milestone: a generic type-safe event system. Good way to close out the templates/lambdas/modern C++ arc with something real.

Then Days 41–43 shifted into data structures and algorithms. Day 41 was Big O complexity analysis with worked examples. Day 42 was implementing a stack and queue from scratch. Day 43 covered binary search and a BST implementation. Classic stuff but doing it in C++ instead of Python forces you to think about memory and ownership in a way that `list.append()` never did.

## What I learned

- Ranges feel like what C++ looks like when it stops fighting you. Chaining `filter | transform | take` reads almost like Python — except it's lazy and zero-cost. Took me a minute to appreciate what that actually means for performance.
- Building a BST with smart pointers was a good callback to the linked list milestone. Ownership semantics with tree nodes are trickier — a node owns its children but doesn't own its parent.
- Big O felt like review coming from Python, but analysing C++ code specifically (iterator invalidation, container choice implications) added a new dimension.

## Next up

- Keep going with data structures — hash maps and graphs are probably next.
- Want to revisit the BST and add balancing (AVL or red-black) once the basics are solid.
