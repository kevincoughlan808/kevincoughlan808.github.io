Kept pushing on the C++ challenge — Days 34–38 this week. This stretch was all templates and modern C++ utilities.

## What I built

Day 34 was function templates and class templates — generic `min`/`max`, template specialisation. Day 35 ramped up fast with variadic templates, fold expressions, and a type-safe `printf`. That one took a while to get compiling but was satisfying once it worked.

Day 36 shifted to `std::optional`, `std::variant`, and `std::any` — the "we have sum types at home" toolkit. Day 37 covered structured bindings and if-init statements, which are small quality-of-life features but immediately made my code cleaner. Day 38 wrapped up with `constexpr` and `consteval`, including a compile-time factorial.

## What I learned

- Variadic templates broke my brain for a bit. Recursive template unpacking feels nothing like Python's `*args` — it's a completely different mental model. Fold expressions help but you still need to understand what the compiler is actually generating.
- `std::optional` feels like it should have existed from day one. Coming from Python where you just return `None` and hope for the best, having an explicit "this might not have a value" type is nice.
- `constexpr` is wild — the idea that you can force computation to happen at compile time instead of runtime still feels like a cheat code. The compile-time factorial example is toy-sized but the implications are obvious.

## Next up

- STL containers and iterators are long overdue — I've been saying "move beyond `std::vector`" for two weeks now.
- Might build a small milestone project that uses templates and `std::variant` together to solidify this stretch.
