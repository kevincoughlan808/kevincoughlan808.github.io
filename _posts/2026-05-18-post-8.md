Big week on the C++ challenge — pushed through Days 18–25, which ended up being a deep dive into memory management.

## What I built

Day 18 covered multiple inheritance and the diamond problem. Virtual base classes make sense but I can see why people say to avoid multiple inheritance unless you really need it.

Day 19 was friend functions, static members, and the factory pattern. Day 20 wrapped up Phase 2 of the Shape hierarchy milestone, which felt good — nice to consolidate everything before moving on.

Then Days 21–25 were all about memory. Started with stack vs heap fundamentals, moved into raw `new`/`delete`, then spent a day deliberately writing broken code — memory leaks, use-after-free, the works. Running AddressSanitizer on my own buggy code was genuinely eye-opening. Finished the week with smart pointers: `unique_ptr`, `shared_ptr`, `weak_ptr`, reference counting, and breaking cycles.

## What I learned

- Intentionally writing buggy code on Day 23 taught me more than any tutorial. Seeing AddressSanitizer catch a use-after-free in real time made the danger feel concrete instead of theoretical.
- After struggling with raw `new`/`delete`, smart pointers feel like a gift. I get why modern C++ pushes you toward them so hard.
- The diamond problem is one of those things that sounds like a contrived interview question until you actually hit the ambiguous member access errors yourself.

## Next up

- Next chunk is probably STL containers and iterators — ready to move beyond `std::vector`.
- Might also look at move semantics more deeply since `unique_ptr` forced me to think about it but I don't feel solid on it yet.
