Another solid week on the C++ challenge. Pushed through Days 12–17, which covered a lot of ground on the OOP side.

## What I built

Day 12 was destructors and RAII — finally understanding why C++ people talk about resource management so much. Day 13 was access control and encapsulation, which felt straightforward coming from Python (though actually enforced this time instead of just vibes and underscores).

Day 14 was operator overloading with a `Vector2D` class — implemented `+`, `-`, `==`, and `<<`. That one was fun. Being able to define what `+` means for your own type still feels a bit magical.

Days 15–17 went deep on inheritance and polymorphism. Built an `Animal` base class with `Dog`, `Cat`, and `Bird` subclasses, then moved into virtual functions, dynamic dispatch, and virtual destructors. Wrapped up with abstract classes and pure virtual functions using a `Shape` hierarchy.

## What I learned

- RAII clicked faster than I expected. The idea that cleanup is tied to scope and destructors makes a lot of things feel safer than manual resource management.
- Virtual destructors are one of those things that seem like a footnote until you realize skipping them causes silent bugs. Glad I hit that early.
- Operator overloading is powerful but I can already see how it could get abused. Just because you *can* overload something doesn't mean you should.
- The jump from basic classes to polymorphism and abstract interfaces felt like the first real "okay, now I'm doing C++" moment.

## Next up

- Templates and generic programming are probably next — that feels like the natural progression.
- Might build another milestone project to consolidate all the OOP stuff before moving on.
